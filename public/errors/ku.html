Errors

en

This document identifies some of the error codes and messages that OMEGA's API return.

This page lists errors by their HTTP status codes as defined in RFC https://tools.ietf.org/html/rfc7231.

Error is follewed standards defined in RFC https://tools.ietf.org/html/rfc7807

  1. UNAUTHORIZED (401)
  2. NOT_FOUND (404)
  3. FOREIGN_KEY (409)
  4. NOT_BIND (422)
  5. VALIDATION_FAILED (422)
  6. INTERNAL_SERVER_ERROR (500)

UNAUTHORIZED (401)

The HTTP 401 Unauthorized client error status response code indicates that the request has not been applied because it lacks valid authentication credentials for the target resource. This status is sent with a WWW-Authenticate header that contains information on how to authorize correctly.

In case of username and password not correctly entered or token not used for viewing a resource this error will emerge, try to Enter your correct username and password

NOT_FOUND (404)

The requested operation failed because a resource associated with the request could not be found. A resource associated with the request could not be found. If you have not used this API in the last two weeks, please re-deploy the App Engine app and try calling it again.

Check the row to find out if it is delted or not! in case of deleted if you have permission you can restore it.

FOREIGN_KEY (409)

In case of entity related to the other entity this error will happened, for inserting or update if the entity not exist this error will shown and for delete if the entity has child prevent of deltion.

NOT_BIND (422)

Not bind means some data sent in the wrong format, for example instead of string, number has been sent. This error mostly happened in the development phase and if end user faced such thing means front-end (website) not works properly, call developer for fix the issue by sending the error code.

VALIDATION_FAILED (422)

Validation failed means some data is not valid. Validations errors are errors when users do not respond to mandatory questions. A validation error occurs when you have validation/response checking turned on for one of the questions and the respondent fails to answer the question correctly (for numeric formatting , required response).

INTERNAL_SERVER_ERROR (500)

The request failed due to an internal error. It may because of database connection or other third party not respond properly. In this case just administrator can fix the issue if error remains
